## Codebase Patterns
- Vault DB migrations: hand-write SQL in `packages/storage/src/db/vault/migrations/<date>_<tag>.sql` and add an entry to `meta/_journal.json` (idx++, when = next sequential timestamp). No drizzle generate command needed.
- The vault DB is per-project (one per vault); no `project_uuid` column needed in vault DB tables.
- New vault DB table → add to `schema.ts` with `sqliteTable`, create accessor functions in `packages/storage/src/suggestion/` (or other relevant subdir).
- Run `bun run codegen` from the WORKTREE root, NOT from the main repo root — the worktrees have independent packages and running from the main repo will regenerate from the wrong source.
- After any API schema change run `bun run codegen` from worktree root (runs `generate-openapi` + `orval` in one step).
- Tests use `createTestApp()` + `seedVault()` pattern; `freshContext.cleanup()` in `afterAll` or at end of test.
- `@hono/zod-openapi`: GET and PUT on the same path (/current) work — but nested path segments (e.g. `/current/{id}` alongside `/current`) don't appear in `getOpenAPIDocument`. Prefer PUT with a JSON body over `POST /current/{id}` when you already have `GET /current`.
