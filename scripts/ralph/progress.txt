## Codebase Patterns
- Command files live in `packages/api/src/commands/fragments/`. They return `{ result, logEntries }` â€” never call storage directly in route handlers.
- Tests use `createTestApp()` + `seedVault()` from `src/__tests__/helpers/`. Command tests get a `CommandContext` via `makeCommandContext()`.
- `splitMarkdown(content, maxHeadingLevel)` returns empty array when headings have no body content. H2+ with level-1 splitting returns entire content as one piece (not zero).
- `deriveKey(rawPiece, existingKeys)` mutates the `existingKeys` set (adds derived key). Must seed set from non-discarded fragment keys (case-insensitive).
- Import schemas are in `packages/api/src/schemas/import.ts`. Route is mounted at `/projects/:projectId/import` via `importRouter`.
- Orval client is regenerated via `bun run --cwd packages/frontend generate` (check orval.config.ts for details).
- Run `bun run verify` from root to typecheck + test all packages.
