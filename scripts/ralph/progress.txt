## Codebase Patterns
- Vault DB migrations: hand-write SQL in `packages/storage/src/db/vault/migrations/<date>_<tag>.sql` and add an entry to `meta/_journal.json` (idx++, when = next sequential timestamp). No drizzle generate command needed.
- The vault DB is per-project (one per vault); no `project_uuid` column needed in vault DB tables.
- New vault DB table â†’ add to `schema.ts` with `sqliteTable`, create accessor functions in `packages/storage/src/suggestion/` (or other relevant subdir).
- Run `bun run codegen` from the WORKTREE root, NOT from the main repo root â€” the worktrees have independent packages and running from the main repo will regenerate from the wrong source.
- After any API schema change run `bun run codegen` from worktree root (runs `generate-openapi` + `orval` in one step).
- Tests use `createTestApp()` + `seedVault()` pattern; `freshContext.cleanup()` in `afterAll` or at end of test.
- `@hono/zod-openapi`: GET and PUT on the same path (/current) work â€” but nested path segments (e.g. `/current/{id}` alongside `/current`) don't appear in `getOpenAPIDocument`. Prefer PUT with a JSON body over `POST /current/{id}` when you already have `GET /current`.

## 2026-06-05 - US-005
- Implemented: margin alignment no longer desyncs when a fragment starts with leading blank lines.
- Root cause: In CM6 (raw/vim) mode, `blockRanges` skips blank lines â€” block 0 is the first non-blank paragraph, which sits below the content origin by the blank-line height. The origin-alignment effect only accounted for the chrome gap between the two scroll containers, not for the internal offset of block 0 within the editor.
- Fix: Read `editorBlocks[0]?.top` in the origin-alignment `useEffect`. Compute `leadingOffset = max(0, block0Top - max(0, delta))` and add it to `rowsPaddingTop`. With no blank lines, `block0Top â‰ˆ max(0, delta)` in steady state, so `leadingOffset â‰ˆ 0` (no regression). With blank lines of height H, `leadingOffset â‰ˆ H`, pushing column row 0 down to align with block 0.
- TipTap mode: unaffected â€” empty paragraphs ARE included as ProseMirror nodes, so block 0 is the first empty paragraph at the content origin.
- Files changed:
  - `packages/frontend/src/components/margins/margin-column.tsx` (origin-alignment effect + `data-testid="margin-rows"`)
  - `packages/frontend/src/components/margins/margin-column.test.tsx` (new test: leading block offset)
  - `specifications/margins.md` (Shipped entry)
- **Learnings for future iterations:**
  - `blockRanges` in CM6 mode is intentionally blank-line-aware (skips them), so `editorBlocks[0].top` is NOT always 0 when there's leading whitespace. Always subtract `max(0, delta)` to get the blank-line surplus â€” don't just add `block0Top` raw (that would double-count the chrome gap).
  - The `data-testid="margin-rows"` attribute on the rows container makes it easy to assert `rowsPaddingTop` (the inline style `paddingTop`) in unit tests.
  - Adding `editorBlocks` to the origin-alignment effect's dependency array is safe: `editorBlocks` only changes on the same triggers as the effect itself (`geometryTick`, `mode`, `fontSize`, `getBlocks`), so no extra re-runs.
