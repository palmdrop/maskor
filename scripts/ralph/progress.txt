## Codebase Patterns
- Settings service lives in `packages/storage/src/settings/`; exported from storage package index
- Bun file I/O: use `Bun.file(path).exists()` to check file existence; `node:fs/promises` for read/write
- Test pattern: `mkdtempSync` for temp dirs, clean up in `afterEach` with `rmSync(dir, { recursive: true, force: true })`
- `DEFAULT_CONFIG_DIRECTORY` in `packages/storage/src/db/registry/index.ts` sets the base config path (`~/.config/maskor`); settings service uses same base path
- All packages use `bun test` for tests; run `bun run verify` for full typecheck + tests
