## Codebase Patterns
- importer package uses mdast-util-from-markdown for AST parsing; `fromMarkdown` returns `Root` with positions on all nodes
- Bun workspace deps: add to package-specific package.json via `bun add` run from project root (Bun handles hoisting)
- Adding a new workspace package dep: edit target package.json directly (`"@maskor/importer": "workspace:*"`), then add path alias to both `packages/<pkg>/tsconfig.json` AND root `tsconfig.json`
- `splitPlainText` and `splitMarkdown` already filter empty pieces; the "empty piece" check in importCommand is purely defensive and won't trigger in normal flow
- `Command<TInput, TOutput>.execute` returns `{ result, logEntries }`; composite commands collect logEntries from inner commands and bubble them up
- `bun run verify` = typecheck + backend tests + frontend tests; run after every story
- `@types/mdast` must be added explicitly to the importer devDependencies; it is not auto-hoisted
- Heading `depth` from mdast is `1|2|3|4|5|6` and maps directly to `HeadingLevel` type
- To extract plain text from mdast heading children: recurse checking `"value" in node` (Text/InlineCode) vs `"children" in node` (Strong, Emphasis, etc.)
- `node.position?.start.offset` gives 0-based byte offset; use `content.indexOf('\n', startOffset)` to find end of heading line
- mammoth has no bundled types and no `@types/mammoth` on npm; it imports fine as-is under Bun (TypeScript uses implicit `any`)
- turndown requires `@types/turndown` as a devDependency; use `{ headingStyle: "atx" }` option for ATX-style headings
- mammoth accepts `{ buffer: Buffer }` input; convert `Uint8Array` with `Buffer.from(input)`
- To strip images in turndown: `td.addRule("stripImages", { filter: "img", replacement: () => "" })`
- `.docx` test fixtures can be generated programmatically with the `docx` npm package (already in workspace)
